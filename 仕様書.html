NORTHEPTION 動画ショート化ツール 仕様書

作成日：2026年6月16日（v1）／最終更新：2026年6月16日（v2）／用途：オバクック等の長尺ゲームプレイ動画から、ショート候補を半自動生成する

v2での変更点（実装後の反映）： ハイライト検出を音声ラウドネス方式とClaude API方式の2モードに対応（--mode loudness / --mode claude、既定はclaude）。 v1で「将来の拡張」としていたClaude API版を正式実装し、既定の検出方式に格上げした。旧方式（ラウドネス）は無料・APIキー不要の選択肢として残している。 詳細は「7. ハイライト検出：Claude APIモード（v2で追加）」を参照。

設計思想：人間の仕事は「判断」だけ

このツールは全自動投稿ツールではない。SNS botと同じ「1対多（機械が候補を出す・最終判断は人）」の動画版。機械が雑務（探す・切る・縦化・字幕）を全部やり、人間に残すのは「どれが良いか選ぶ」判断だけにする。

Before（手作業）After（このツール導入後）
1時間の動画をスクラブして良い瞬間を探す → 手で切る → 縦にリサイズ → 字幕を打つ → 結局やらない 動画を1本フォルダに置いてコマンド1つ → 縦化・字幕済みの候補が6〜8本出てくる → 短い候補を見て1本選ぶだけ

人間がやること（これだけ）

  1. 選ぶ：出力された候補クリップ（各15〜30秒・縦・字幕付き）を見て、出す1本を選ぶ
  2. 承認：キャプション（SNS botで生成）を確認し、双子が映る場合は保護者とすり合わせた上で手動投稿
探す・切る・縦化・字幕は全部ツールが自動でやる。人間の所要は「短い候補を数本見て選ぶ」＝数分。

全体フロー

① 1時間の動画（input.mp4）を /input フォルダに置く
② コマンド実行（例：npm run clips。既定は--mode claude）
③ ハイライト検出（モードにより分岐）
　・--mode claude（既定）：動画全体をWhisperで文字起こし → タイムスタンプ付きテキストをClaude APIに渡し、盛り上がり区間をN個選んでもらう
　・--mode loudness：音声ピーク（声・笑い）の時間帯を自動抽出 → 候補タイムスタンプN個
④ 各候補を FFmpegで切り出し（15〜30秒）
⑤ 9:16縦化（背景ぼかしレターボックス）
⑥ 各クリップをfaster-whisperで個別に文字起こしして字幕生成 → クリップに焼き込み
⑦ /candidates に clip_01.mp4 … clip_0N.mp4 ＋ manifest.json を出力
⑧ 人間が候補を見て1本選ぶ（以降は手動投稿）

技術スタック

項目内容
実行環境ローカルPC（Windows）。動画処理は重くFly.io無料枠に不向きなため、ローカルでバッチ実行する。SNS botは従来通りFly.io。
オーケストレーションNode.js（既存スタックに合わせる）。各CLIを child_process で呼び出す
動画処理FFmpeg（カット・縦化・字幕焼き込み・音声解析）。オープンソース・無料
文字起こし実装はfaster-whisper（pip）。当初予定のwhisper.cppはWindowsビルドが大変なため、仕様書記載の代替案として採用した
ハイライト検出2モード対応（--modeで切替、既定claude）
・claude：全体文字起こし＋Claude API（claude-sonnet-4-6）で盛り上がり区間を判定。精度優先
・loudness：音声ラウドネス時系列のピーク検出。完全無料・ローカル・APIキー不要だが誤検出が出やすい
コスト--mode claude（既定）は実行ごとに少額のAPI課金が発生（数円規模）。--mode loudnessなら0円

各ステップの実装指示

1. 入力

  • /input/input.mp4 を処理対象とする（ファイル名は引数で上書き可）
  • 作業用に音声だけ抽出：ffmpeg -i input.mp4 -ac 1 -ar 16000 -vn work/audio.wav（whisper.cpp用に16kHzモノラル）

2. 音声ピーク検出（候補タイムスタンプの抽出）（--mode loudness 時のみ。既定は8番のClaude APIモード）

盛り上がりの代理指標としてラウドネス（音量）の時系列を使う。掛け合いゲーム（オバクック2人プレイ）では声が上がる瞬間＝面白い瞬間と相関しやすい。

  • FFmpegの ebur128 フィルタで瞬時ラウドネス（Momentary, M値）を出力し、stderrから時刻＋M値をパースして1秒粒度の時系列を作る
    　例：ffmpeg -i work/audio.wav -af ebur128=metadata=1 -f null - 2> loudness.log → ログを正規表現でパース
  • 移動平均でノイズを均し、上位ピークを検出。近接するピークは1つの区間にマージ
  • 各ピークを中心に 15〜30秒の窓に広げる（前2〜3秒・後ろ長めにして"オチ"を含める）。窓が重なる候補は統合
  • ラウドネススコア上位から N件（既定8件）を候補として採用
限界（重要）：音声ピークは「うるさい＝面白い」ではない。ゲームSEや苛立ちの声も拾う。誤検出が出る前提で候補を多めに出し、最終選択は人間が行う（＝このツールの設計思想そのもの）。

3. クリップ切り出し（FFmpeg）

ffmpeg -ss [開始秒] -i input.mp4 -t [長さ秒] -c:v libx264 -c:a aac work/cut_NN.mp4

※ -ss を入力前に置くと高速シーク。フレーム精度が必要なら入力後に置く。

4. 9:16縦化（背景ぼかしレターボックス）

オバクックは俯瞰固定画面で、単純な中央クロップだと厨房の端が切れる。元映像をぼかして背景に敷き、その上に元映像全体を載せる方式が安全。出力は1080×1920。

ffmpeg -i work/cut_NN.mp4 -filter_complex \
"[0:v]scale=1080:1920:force_original_aspect_ratio=increase,crop=1080:1920,boxblur=20:5[bg]; \
 [0:v]scale=1080:-1[fg]; \
 [bg][fg]overlay=(W-w)/2:(H-h)/2" \
-c:a copy work/vert_NN.mp4

5. 字幕生成・焼き込み

  • 各クリップの音声を whisper.cpp に渡し .srt を生成（日本語モデル：ggml-medium 目安。精度優先なら large、速度優先なら small）
  • SRTをFFmpegで焼き込み：
    ffmpeg -i work/vert_NN.mp4 -vf "subtitles=work/sub_NN.srt:force_style='Fontsize=18,Outline=2,Alignment=2'" candidates/clip_NN.mp4
  • 字幕スタイルは下部中央・縁取り付き（音声オフ視聴対策）。フォントはNoto Sans JP等の日本語対応を指定
限界：ゲームSE混じりの音声では字幕に誤変換が出ることがある。人間は字幕が綺麗な候補を選べばよい（or 採用1本だけ手直し）。

6. 出力

  • /candidates/clip_01.mp4 … clip_0N.mp4（縦・字幕済み・即視聴可）
  • /candidates/manifest.json：各候補の { ファイル名, 元動画の開始/終了秒, 長さ, ラウドネススコア, 字幕テキスト }。人間が一覧で見て選びやすくするため

7. ハイライト検出：Claude APIモード（v2で追加・既定）

ラウドネス検出（うるさい＝面白い、という乱暴な代理指標）の誤検出を減らすため、文字起こしの内容そのものをClaudeに判断させる方式。仕様書v1の「将来の拡張」案を正式実装し、既定モードに格上げした。

  • 動画全体の音声を抽出し、faster-whisperで全文文字起こし（タイムスタンプ付き）
  • 文字起こしテキストを「[開始-終了] テキスト」形式に整形し、Claude API（モデル：claude-sonnet-4-6）に渡す
  • プロンプト例：「以下はゲーム実況の文字起こしです。2人の掛け合い・笑い・驚き・盛り上がりが伝わる区間をN箇所選び、各区間の開始秒・終了秒・選んだ理由をJSONで返してください」
  • 返ってきたJSON（[{"startSec":..,"endSec":..,"reason":".."}, ...]）のタイムスタンプを使い、以降は3〜6番の手順（切り出し・縦化・字幕焼き込み）を共通実行
  • 選定理由（reason）は manifest.json に格納し、人間が候補を選ぶ際の参考情報にする
注意：このモードはClaude APIを呼ぶため、APIキー（ANTHROPIC_API_KEY）が必要かつ実行ごとに少額の課金が発生する（v1で「完全無料」としていた前提から外れる）。APIキーはプロジェクト直下の .env ファイル（ANTHROPIC_API_KEY=sk-ant-...）で管理し、OneDrive経由で開発環境間を引き継ぐ。Windowsレジストリ等PC単体にしか効かない保存方法は使わない。課金やAPIキー設定を避けたい場合は --mode loudness（v1のラウドネス方式）に切り替えれば従来通り無料・ローカルのみで動作する。

前提・初回セットアップ（一度だけ）

セットアップ自体もClaude Codeに実行させる。以降は「動画を置く→コマンド」だけ。

  • FFmpeg をインストールしPATHを通す
  • whisper.cpp をビルド（or プレビルドバイナリ）＋ 日本語対応 ggml モデル（medium）をダウンロード。Windowsでビルドが詰まる場合は pip install faster-whisper に切替
  • Node.js（既存環境）
  • 日本語フォント（字幕焼き込み用）
PC性能の確認：Whisperのローカル文字起こしはCPU/GPUに依存。1時間動画の全文字起こしは重いが、本ツールは切り出した短いクリップ（合計数分）だけを文字起こしするので負荷は小さい。初回に1クリップで所要時間を計測し、モデルサイズを調整する。

正直な限界と、それを判断で吸収する設計

  • （--mode loudness時）音声ピーク検出は「面白い」を保証しない（うるさい瞬間を拾う）。→ 候補を多め＋人間が選ぶことで吸収
  • （--mode claude時）Claudeの判断は文字起こし（faster-whisper）の精度に依存する。文字起こし自体が誤っていれば、Claudeもその誤った内容を基に判断する。また実行ごとに少額の課金が発生し「完全無料」ではなくなる
  • 字幕は誤変換が出る。→ 綺麗な候補を選ぶか採用1本だけ手直し
  • クラウドで「全自動・無料・透かし無し」は実在しない。無料を貫くなら--mode loudness＋人の最終確認がこのツールの条件

NORTHEPTION運用ルールとの整合

  • X APIは使わない／投稿は手動（SNS botと同じ。発信の最終判断は人間）
  • 枠：金枠（slot=story／居着かせ）。認知の入口ではない
  • ハッシュタグ例外：オバクックはSF6ではないため、この動画投稿は #SF6 を外し #NthWIN ＋ #Overcooked2 等に差し替える
  • 未成年ゲート：双子（14歳）が映る場合、投稿前に保護者（@murasaki_15922）とすり合わせ
  • IP：Overcooked! 2 のショート投稿・収益化はTeam17より許諾済み（公式チーム垢・選手個人垢いずれも可。ゲーム本体の販売を伴わないこと）
