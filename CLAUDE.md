# このプロジェクトでの個別ルール

- `input/` `work/` `candidates/` など、ユーザーの実ファイルが置かれる可能性があるディレクトリに対して、
  `rm -rf xxx/*` のような一括削除を絶対に行わない。
  - **理由**: 過去に動作確認用の合成テスト動画を消すつもりで `rm -rf work/* candidates/* input/*` を実行し、
    ユーザーが`input/`に置いていた実際の動画ファイル（input.mp4）を誤って削除してしまったことがある。
  - 自分（Claude）のテスト用ファイルは、可能な限り別のサンドボックス/一時フォルダに置く。
  - 実フォルダを使ってテストする場合は、自分が作成した特定のファイル名だけを削除する（フォルダ全体のglob削除はしない）。
  - 削除前にユーザーのファイルかどうか判断できない場合は、削除せず確認を取る。
  - **`input/input.mp4` に自分のテスト用合成動画を直接書き込まない（`-y`上書きも含む）。**
    過去に機能テスト中、`ffmpeg ... -y input/input.mp4` で合成動画を直接書き込み・上書きし、
    最後にテスト後片付けで削除したことで、その間にユーザーが置いた本物の動画が
    消えた可能性がある事象が発生した。自分のテスト動画は `work/_test_fixtures/` など
    `input/` 以外の場所に生成し、`input/input.mp4` には一切書き込まない。
